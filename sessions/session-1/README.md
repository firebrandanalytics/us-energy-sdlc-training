# Session 1 — From Autocomplete to Agentic

U.S. Energy AI Software-Development Training · 5 × 2-hour online course · **Session 1 of 5**

> **Theme: From Autocomplete to Agentic. The mindset shift, and why prep work is the work.**

This first session is conceptual. We are not in Claude Code yet — that starts in
Session 2. The goal today is to change how you *think* about working with an AI
agent, because the habits you bring decide whether the rest of the course pays
off. There is one short live demo and one short hands-on at the end; everything
else is discussion.

No dataset, no setup, nothing to install for today. Just come curious.

---

## Where this session sits

The course runs as one connected arc. You will use an AI agent to **comprehend**
unfamiliar data and code, **scope** and **direct** the work, **sharpen** a vague
request into a real specification, then **extract → plan → build** a clean data
pipeline and **scale** it with parallel agents. Session 1 is the mindset that
makes all of that work.

## What we cover today

1. **The mindset shift** — moving from tab-complete coding (the AI types, you
   read) to agentic work (you direct, the AI executes).
2. **What an AI agent actually is** — it plans, uses tools, iterates, and
   self-corrects, and why that is a different thing from a chat assistant.
3. **Your role stays human** — you define what "done" looks like, set the
   constraints, review before approving, and own the outcome.
4. **Calibrating trust by risk** — how much to lean on the agent depends on how
   bad it is if it's wrong. Short group discussion.
5. **The core reframe** — stop asking how the AI does what you do; start
   structuring the environment so it performs at its best.
6. **The three things AI is reliably weak at** — and the practical fix for each.
7. **Live demo** — the same data task run from a weak prompt versus a sharp,
   AI-ready spec.
8. **Short hands-on** — turn a vague data request into an AI-ready spec.

---

## The one idea to leave with

If you take a single thing from today, take this:

> **The agent is fast and confident. You are the judgment it doesn't have. Your
> highest-leverage move is no longer the keystroke — it's defining the work
> clearly and reviewing the result.**

Everything in this course is built on that. The skill we start practising today —
turning a vague ask into something precise enough to hand off — is the same skill
you will use in Session 4 to turn *"give me clean monthly volumes by terminal"*
into a pipeline that produces the right numbers.

## Looking ahead to Session 2

Session 2 is hands-on — you will run Claude Code yourself. Two things to line up
**before** then:

- Get your **Claude Code access working**. We will verify it live at the top of
  Session 2, but the session moves faster if it's already going for you. Flag any
  access problems now.
- Come with **one real task from your actual work** in mind — a small, concrete
  thing you'd genuinely like done (a query to write, a script to clean up, a file
  to reorganise). Not a toy problem. We'll point the agent at it together, and it
  becomes the basis for Homework #1.
